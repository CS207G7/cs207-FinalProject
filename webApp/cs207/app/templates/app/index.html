TOOLKIT

This chemkin is a collection of algorithms aimed at predicting the time evolution of species concentration, finding the rate of change of a chemical species and calculating the rate of change of a certain specie. The toolkit is designed for flexibility, portability, easiness of use and easiness of extension. Its software design follows an object-oriented approach and its code is written on Python.

Main Utilities

Parse Reactions from XML

parsed = chemkin.ReactionParser('path_to_reaction_xml')
The method chemkin.ReactionParser() parses XML files and returns the details of the reactions contained like species name, reaction type, reaction's concentration, and temperature.

Wrap Parsed Reactions into Reaction Class

reactions = chemkin.Reaction(parsed, T)
The method chemkin.Reaction() wrap the reactions' detail into a Reaction Class at a given Temperature (T).

Obtain Reaction Components

V1, V2 = reactions.reaction_components()
The method reactions.reaction_components() stacks each v1 into V1, and each v2 to V2 for each reaction.

Obtain Reaction Coefficients

k = reactions.reaction_coeff_params()
The method reactions.reaction_coeff_params() returns the reaction coefficients both for Reversible and Irreversible reactions. Furthermore it returns the coefficient accordingly with the given datasets. For instance, if Arrhenius is found it checks whether using the standard or the modified version.

Obtain Reaction Rates

rr = chemkin.ChemKin.reaction_rate(V1, V2, X, k)
The method chemkin.ChemKin.reaction_rate() return the reaction rate for the given concentration (X) and reaction coefficient (k) of the set of species involved in the reaction as reactants (V1) and products (V2).

    >>> from kinetics.chemkin import ReactionParser

    >>> from kinetics.chemkin import ChemKin

    >>> T = 750

    >>> X = [2, 1, 0.5, 1, 1 ,0.5, 0.5, 0.5]

    >>> V1, V2 = reactions.reaction_components()

    >>> k = reactions.reaction_coeff_params()

    >>> rrs = ChemKin.reaction_rate(V1, V2, X, k)

Add'l Feature: History

Motivation and Feature Description

FIND REACTIONS... QUICK
Our team developed a feature able to keep track of all the elementary reactions computed by a user. The feature efficiently browses into a list of reactions that have been computed and it sorts them by elements of interest such as species, temperature, date, and reversible or non-reversible reactions. We think that the feature could be really useful for scientists interested in building their own database of elementary reactions. Indeed, the devised feature allow the user to explore reactions without the need for computing again. (Beside saving time for the computation, the user might grow its own database that could be eventually shared with other users). Our feature ultimately allows the user to find quickly the information related to the reactions already computed.

Feature and Code Base

The feature fits into our code base because it stores each reaction computed by chemkin. Our feature stores them in a database adjacent to the one with NASA polynomials. The code allows the user to find reactions choosing among different element of interests using the methods enfolded in the module section where it is explained how do they work.

Module

History.py
Our team has built a module history.py. It handles the results of each reactions computed by the user and It also stores them into a sqlite3 database. Thanks to the module History.py, the user is able to access the information contained in the database using the methods that follows.

Methods

get_type()

This method is enabled by inputting the type(i.e. reversible or not reversible) of the reaction. It returns the list of reactions of similar type.

get_specie()

This method is enabled by inputting species of the reaction (e.g. H, H2, O2, etc.). It returns the list of reactions computed with the same species.

get_temperature()

This method is enabled by inputting temperature of the reactions. It returns the list of reactions computed with the same temperature.

get_by_date()

This method is enabled by inputting the date on which the reaction has been computed. It returns the list of reactions computed in the same date.

User's Experience

The user can query any information contained in the reactions that have been computed. For instance, the user can find all the reactions sort by elements of interest (e.g. type, species in the reactions, temperature of the reactions, etc).
